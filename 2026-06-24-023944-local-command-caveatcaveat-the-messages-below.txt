> TraceSurgeon                                                                  
  Causal Provenance & Dependency-Aware Debugging for AI Agents                  
  TraceSurgeon is a research-grade diagnostic SDK designed to solve the         
  "Attribution Washout" problem in autonomous agents. While standard            
  interpretability tools can attribute an LLM's output to specific text spans,  
  they are "dependency-blind"—they cannot distinguish between a downstream      
  reasoning symptom and an upstream factual cause.                              
  TraceSurgeon introduces Recursive Attribution Redistribution to trace agent   
  failures back through the execution DAG to the original poisoned observation  
  or tool output.                                                               
                                                                                
  1. The Core Insight                                                           
  In complex agents (like those built with LangGraph), failures are rarely      
  contained in a single step. An agent takes a "Bad Action" because its         
  "Reasoning" was flawed; however, that reasoning was only flawed because it    
  inherited "Poisoned Data" from an earlier tool output.                        
  Current SOTA attribution (e.g., ContextCite, NeurIPS 2024) suffers from Blame 
  Hoarding: it attributes 90%+ of the failure to the final reasoning step       
  because that step has the highest linguistic correlation with the error.      
  TraceSurgeon uses the agent's execution graph to mathematically "flow" that   
  blame backward to the true root cause.                                        
                                                                                
  2. Validated Research Wedge                                                   
  Empirical testing on Qwen-2.5-3B using adversarial agent trajectories         
  confirmed:                                                                    
                                                                                
  0% Localization Accuracy: Vanilla sparse-linear surrogates (Lasso)            
  consistently failed to identify upstream poisoned tool outputs.               
  100% Blame Hoarding: 100% of failures were misattributed to downstream        
  reasoning nodes.                                                              
  The Logit Trap: Reasoning steps produce log-probability drops up to 6.8x      
  higher than factual observations, effectively "cloaking" the root cause from  
  existing tools.                                                               
                                                                                
                                                                                
  3. Technical Architecture

      subgraph "I. Instrumentation Layer"                                       
          A[LangGraph Node] -->|@ts.step| B[State Delta Interceptor]            
          B --> C[OpenInference Trace Exporter]

  4. Key Engineering Features                                                   
                                                                                
  LangGraph Native Hook: Automatically extracts execution traces by             
  intercepting State transitions and Checkpoints, requiring zero manual logging 
  from the developer.                                                           
  Semantic Edge Inferrer (SEI): Uses a local Cross-Encoder (ms-marco-MiniLM) to 
  discover hidden dependencies between steps even when explicit variable names  
  change.

  in long-horizon traces (50+ steps).

  Is it Useful? Yes — the problem is real.

  high-entropy reasoning steps. This problem will grow worse as agent graphs get

  TraceSurgeon — Improved Assessment (Research-Grounded)

  The Problem Is Verified Real

  The research confirms this isn't a manufactured gap:

  - Multi-agent LLM systems show failure rates up to 86.7% in real deployments
  - SOTA reasoning LLMs achieve <10% accuracy on failure attribution tasks in

  - LLM-as-Judge approaches score <10% accuracy on lengthy agent logs — the
  current "just ask GPT to debug it" approach is nearly useless
  - No existing observability tool (LangSmith, OpenInference, Phoenix) claims or

  TraceSurgeon is addressing a confirmed, quantified gap.

  ---
  The Competitive Landscape Changed — You Have Real Rivals

  The research surfaced three direct competitors you must address:

  🔴 AgentTrace (arxiv 2603.14688) — Your Closest Rival

  - Reconstructs causal graphs directly from execution logs, no LLM inference at
  debug time
  - Claims high accuracy and sub-second latency on root cause localization
  - Outperforms heuristic and LLM-based baselines significantly
  - Critical difference from TraceSurgeon: no gradient methods, no SIG — purely
  log-based graph reconstruction

  🟡 CHIEF Framework (arxiv 2602.23701)

  - Uses hierarchical causal graphs + counterfactual attribution
  - Achieves 77.59% agent-level accuracy, 29.31% step-level accuracy on failure
  logs
  - ~2.5–3x token cost of direct prompting
  - Gap: step-level accuracy is poor, which is exactly where TraceSurgeon claims
  to excel

  🟡 MP-Bench (arxiv 2603.25001)

  - First benchmark for multi-perspective failure attribution
  - Surfaces that existing methods assume a single deterministic root cause,
  missing real-world ambiguity
  - TraceSurgeon should adopt or cite this benchmark for credibility

  1. Drop "SIG Requires Gradients" as a Core Method → Replace with Hybrid

  The research confirms LIME/SHAP for black-box LLMs is viable but expensive

  proxy model framework achieves >90% fidelity at 11% of the cost (88.2%
  reduction, arxiv 2505.12509).

  ─────────────────────────────────────────
  Open-weight (Qwen, Llama, Mistral)  →  Shapley-Integrated Gradients (current)

● TraceSurgeon — Complete Build Roadmap

  ---

  - [ ] LangGraph callbacks deep-dive — specifically on_chain_start,

  - [ ] Git + GitHub — commit every working step

  Output: A toy LangGraph agent (search + summarize) that you fully understand
  internals of. This becomes your test bed for everything else.

  ---
  Phase 1: Trace Capture (Week 3–4)

  The most important phase. Everything else depends on this.

  Build:
  LangGraph Agent
      ↓  (LangGraph BaseCallbackHandler)
  TraceInterceptor
      ↓
  JSON trace file:  { step_id, node_name, inputs, outputs, timestamp,
  parent_step_id }

  Concrete tasks:
  - [ ] Subclass BaseCallbackHandler — override the 6 key methods
  - [ ] Assign a unique ID to every node execution
  - [ ] Record parent_step_id so you know what called what
  - [ ] Save the full trace to a .jsonl file after every run
  - [ ] Write a function that loads the trace and prints it as a tree

  Milestone: Run your test agent, look at the .jsonl, and see the full execution
  story written out.

  ---
  Phase 2: DAG Builder (Week 5–6)

  Turn the flat trace into a real graph.

  Build:
  trace.jsonl  →  NetworkX DiGraph  →  adjacency list with blame weights (all
  1.0 for now)

  Concrete tasks:
  - [ ] Parse .jsonl into a networkx.DiGraph
  - [ ] Each node stores: { name, inputs, outputs, duration_ms, success: bool }
  - [ ] Each edge stores: { data_passed } (what output became the next node's
  input)
  - [ ] Write find_failure_path(graph, failed_node_id) — returns the chain of
  ancestors
  - [ ] Visualize with matplotlib or pyvis — just get it on screen

  Milestone: Run a failing agent, call find_failure_path(), and see the chain of
  nodes that led to the failure printed out correctly.

  ---
  Phase 3: Blame Scorer v1 (Week 7–8)

  This is what makes TraceSurgeon more than just a logger.

  Build a simple heuristic scorer — no ML yet:
  def score_blame(node, failed_node):
      score = 0
      score += 0.4 if node.output_was_used_directly_by(failed_node) else 0
      score += 0.3 if node.is_tool_call else 0          # tool outputs are 
  suspicious
      score += 0.2 if node.output_contains_error_signal else 0
      score += 0.1 * (1 / graph_distance(node, failed_node))
      return score

  Concrete tasks:
  - [ ] Define "failure" — node returns an exception, or final output contains a
  known bad pattern
  - [ ] Score every ancestor node using the heuristic above
  - [ ] Sort ancestors by blame score, return top 3
  - [ ] Write a blame_report(trace_path) function that outputs ranked suspects

  Milestone: Intentionally poison one tool's output in your test agent.
  blame_report() should rank that tool as the #1 suspect.

  ---
  Phase 4: CLI + Basic UI (Week 9–10)

  The key accuracy principle: I stopped trusting LangGraph's parent links and 
  reconstruct data flow from timestamps instead. That's what makes it
  version-proof and loop-safe.

  Honest limitations that remain

  These are real and worth knowing before you trust it on production traces:

  1. Wrong-but-not-error answers — if a tool returns plausible-but-incorrect
  data with no error keywords, text detection won't catch it. Only Phase 6 
  (counterfactual) can. This is a fundamental limit of every attribution tool,
  per your research.
  2. True parallel branches — I linearize concurrent nodes by start-time. Blame
  still works via the ancestor cone, but the visual order is approximate.
  3. Error-keyword false positives — a node legitimately discussing "the error
  rate" could trip detection. The introducer/tool signals dampen this, but it's
  heuristic.
